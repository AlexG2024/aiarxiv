# arXiv AI Digest for Telegram

MVP-проект для Telegram-канала, который автоматически собирает дневную очередь AI-статей с arXiv и публикует их по расписанию в формате, понятном обычному читателю.

## Что делает проект

- получает свежие статьи из AI-категорий arXiv;
- отбирает статьи, которые ближе к практическому применению и интересны обычному пользователю;
- собирает дневную очередь постов один раз в день;
- публикует посты в Telegram-канал по расписанию;
- не публикует одну и ту же статью повторно.

## Правила постинга

Проект ориентирован не на "все подряд статьи", а на материалы, которые можно объяснить обычному читателю простым языком.

В очередь попадают статьи, которые:

- относятся к AI-категориям arXiv;
- ближе к прикладным сценариям, чем к чистой теории;
- можно связать с пользой для человека, работы, учебы или повседневного использования AI;
- не были опубликованы в канале раньше.

Дополнительно пайплайн:

- усиливает статьи по сильным consumer-friendly паттернам;
- штрафует слишком академичные и узкоспециализированные темы;
- хранит историю публикаций и не постит один и тот же `arXiv id` повторно.

## Формат поста

Каждый пост собирается в таком виде:

- русский заголовок;
- оригинальный английский заголовок курсивом;
- короткое вводное описание;
- `Что сделали`;
- `Какие проблемы решает`;
- `Где пригодится`;
- `Рекомендация`;
- `Вывод`;
- `Детали`;
- ссылки на `arXiv` и `PDF`.

## Расписание публикаций

Проект работает через дневную очередь.

- ночью собирается очередь из `10` постов на текущий день;
- очередь создается несколько раз с запасом, чтобы переживать задержки GitHub Actions;
- затем отдельный workflow проверяет очередь каждые `10` минут;
- затем отдельный workflow проверяет очередь каждые `10` минут;
- если пост уже должен выйти, публикуется только `1` следующий due-пост.

Текущее расписание по Москве:

- сборка очереди: `00:10`, `00:30`, `00:50` и далее каждые `20` минут до `06:50`;
- окно публикаций: с `08:10` до `17:10`;
- poller догоняющей публикации: каждые `10` минут с `08:00` до `20:50` и еще один контрольный запуск в `21:00`;
- план: `10` постов в день, по одному в час.

Если GitHub Actions задерживается, система не ломает расписание полностью:

- дневная очередь не строится заново в течение того же дня;
- старые очереди сбрасываются на следующий день;
- poller публикует только один следующий пост за запуск, без массового "залпа".

## Категории arXiv

По умолчанию проект использует:

- `cs.AI`
- `cs.LG`
- `cs.CL`
- `cs.CV`

## Структура

- `main.py` - ручной единичный прогон старого пайплайна
- `build_daily_queue.py` - сборка дневной очереди
- `publish_due_post.py` - публикация одного запланированного поста
- `weekly_digest.py` - генерация weekly digest
- `src/config.py` - чтение настроек
- `src/arxiv_client.py` - загрузка статей из arXiv
- `src/filters.py` - отбор статей, ближе к обычному читателю
- `src/formatter.py` - сборка текста поста
- `src/telegram_client.py` - отправка сообщений в Telegram
- `src/storage.py` - хранение уже опубликованных статей
- `src/pipeline.py` - общий сценарий работы
- `data/posted.json` - база опубликованных arXiv id
- `data/history.json` - история опубликованных статей
- `data/daily_queue.json` - очередь постов на текущий день
- `.github/workflows/build_daily_queue.yml` - ночная сборка очереди
- `.github/workflows/publish_due_post.yml` - частый poller публикации

## Что нужно сделать вам

### 1. Создать Telegram-бота

Через `@BotFather`:

1. Открыть `@BotFather`
2. Выполнить `/newbot`
3. Придумать имя и username бота
4. Сохранить `BOT_TOKEN`

### 2. Создать канал

1. Создать Telegram-канал
2. Добавить бота в канал
3. Выдать боту права администратора на публикацию сообщений

### 3. Подготовить переменные

Нужно знать:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

В `TELEGRAM_CHAT_ID` можно указать:

- `@channel_username`
- или числовой `chat_id`

## Локальный запуск

### 1. Создать `.env`

Скопировать `.env.example` в `.env` и заполнить значения.

### 2. Запустить

```powershell
py -3 build_daily_queue.py
py -3 publish_due_post.py
```

### Режим теста

Если не хотите сразу постить в канал:

```powershell
$env:DRY_RUN='true'
py -3 main.py
```

В этом режиме скрипт только покажет в консоли, что он собирался бы опубликовать.

## GitHub Actions

Для автозапуска через GitHub Actions добавьте в Secrets репозитория:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

Workflow'ы настраиваются отдельно:

- `build_daily_queue.yml` - несколько попыток ночью, пока очередь на день не соберется
- `publish_due_post.yml` - частый запуск, который публикует только один due-пост

Что важно:

- GitHub Actions может запускаться с задержкой, поэтому публикация построена через дневную очередь и частый poller, а не через один "точный" запуск в час;
- файлы `data/posted.json`, `data/history.json` и `data/daily_queue.json` обновляются самим workflow и коммитятся обратно в репозиторий;
- после первого пуша нужно открыть вкладку `Actions` и убедиться, что workflow разрешены для репозитория.

## Ограничения MVP

- без LLM текст постов будет шаблонным;
- arXiv не дает готовый рейтинг статей;
- отбор "интересно обычному читателю" сейчас основан на простых правилах и ключевых словах.
